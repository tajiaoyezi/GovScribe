# c05 高频文种样稿评测集

建立日期：2026-06-19

## 评测口径

- 范围：覆盖 c05 深做档 9 个文种，每个文种选一个代表性子类建立固定样稿。
- 样稿性质：以下样稿为脱敏占位的种子评测样稿，用于验证 rubric 可重复打分；不代表真实客户数据、真实模型效果或最终 UAT 采纳率。
- 评分来源：对齐 PRD「验收 rubric 草案」的四维：文种规范、结构完整、行文关系、机关口径。
- 采纳口径：采纳状态只允许四档：直接用 / 小改 / 大改 / 弃用；直接用 + 小改计入采纳。
- 记录位置：逐篇评分记录见 `docs/other/c05-high-freq-doctype-evaluation-records.csv`。
- 边界：真实范文与目标模型实测校准属于 7.3 / 8.1；本文件只建立可重复评分的评测集与初始评分记录。

## 评分锚点

| 维度 | 5 分 | 3 分 | 1 分 |
| --- | --- | --- | --- |
| 文种规范 | 文种选择、标题、称谓 / 主送、落款均符合该文种规则 | 文种正确但局部要素不完整或表述松散 | 文种误判或核心格式错误 |
| 结构完整 | 标准段落结构与必备要素完整，缺项以占位提示 | 主体结构可用但缺少 1 个关键要素或顺序不清 | 多个关键要素缺失，无法据此核稿 |
| 行文关系 | 上行 / 下行 / 平行口吻明确，措辞与对象匹配 | 口吻大体可用但存在轻微模板腔或对象感弱 | 口吻方向错误，例如请示写成通知 |
| 机关口径 | 政治性表述稳妥，不臆造数据 / 文号 / 单位名，敏感项留占位 | 基本稳妥，但有少量需核稿人改写的泛化表述 | 出现臆造事实、敏感风险或自动发文语义 |

## 复评步骤

1. 固定使用本文件中的 `sample_id` 与样稿文本，评分人不得在评分前改写样稿。
2. 对每篇样稿分别填写四维 1-5 分；若评分人需要修改样稿才能采纳，采纳状态填“小改”或“大改”。
3. 采纳率统计只读取 `docs/other/c05-high-freq-doctype-evaluation-records.csv` 中的 `adoption_status`：直接用 / 小改计入采纳，大改 / 弃用不计入。
4. 若后续替换为真实模型输出，必须保留原 `sample_id` 或新增版本号，避免不同样稿混入同一评分记录。
