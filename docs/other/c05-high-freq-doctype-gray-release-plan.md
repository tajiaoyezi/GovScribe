# c05 高频文种灰度放开计划

建立日期：2026-06-19

## 计划口径

- 依据：`docs/other/c05-high-freq-doctype-sample-inventory.md` 的样本盘点结论，以及 `docs/other/c05-high-freq-doctype-evaluation-set.md` 的 rubric 评测集。
- 灰度目标：先开放样本可证明充足的深做文种 / 子类给试点文秘，按文种规范、结构完整、行文关系、机关口径四维评分，并按直接用 / 小改 / 大改 / 弃用四档采纳状态打标。
- 首批原则：只以仓库内可核验样本数量作为首批依据。当前首批候选不是 design 示例中的通知 / 请示 / 报告，而是 1.1 盘点中已登记且 `>=100` 的方案子类。
- 边界：本计划只定义 c05 灰度放开顺序与打标表，不写入 c06 文种能力档 / 标黄配置，不替代 c08 的真实采纳决策记录，不声称已完成真实 UAT。

## 灰度准入门槛

每一批进入试点前必须同时满足：

- 原文样本已到位，并完成清洗 / 脱敏入库。
- c03 可按文种 / 子类检索到可用样例，并能返回样例不足标记。
- c05 已具备对应结构契约、few-shot 拼装与 SSE 生成路径。
- c04 `draft.create` 授权已对试点文秘放行，未授权用户仍 fail-closed。
- 每篇灰度稿按 7.1 rubric 四维评分，并登记四档采纳状态。

## 打标字段

逐篇灰度记录必须至少包含：

- `batch_id`
- `doctype`
- `subtype`
- `sample_id` 或真实生成稿引用
- 四维评分：`doctype_norm_score`、`structure_score`、`direction_score`、`organ_tone_score`
- `adoption_status`：直接用 / 小改 / 大改 / 弃用
- `counts_as_adopted`：直接用 / 小改为 `true`，大改 / 弃用为 `false`
- `reviewer`
- `reviewed_at`
- `notes`

批次跟踪表见 `docs/other/c05-high-freq-doctype-gray-release-batches.csv`。其中当前的 `seed_adoption_status` 来自 7.1 种子评测集，仅用于验证打标字段可用；真实试点采纳状态须由后续 c08 在线核稿 / 采纳决策记录承接。
