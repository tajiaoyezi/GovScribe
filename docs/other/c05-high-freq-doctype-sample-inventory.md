# c05 高频文种范文样本盘点

盘点日期：2026-06-19

## 盘点口径

- 范围：c05 `c05-high-freq-doctypes` 覆盖的 9 个高频文种：通知、请示、报告、函、会议纪要、通报、批复、讲话稿、方案。
- 数量口径：仅采信仓库内已提交材料中明确写出的范文数量；模板条目、标题示例、链接示例不折算为样本量。
- 充足线：对照 PRD 风险口径，`<100` 条为稀缺线；每文种宜 `>=100` 篇才具备深做灰度底气。若只登记到子类数量，则按子类分别判断，文种整体标为“部分充足”。
- 边界：本盘点仅作为 c06 文种分流与 c05 灰度放开顺序输入；不在 c05 写入文种 / 子类能力档、不判定是否标黄、不替代 c06 分级表。

## 证据来源

- `docs/prds/intelligent-gov-document-agent.prd.md`：明确要求开工前按 9 个高频文种盘点客户存量范文数量、到位时间、清洗 / 脱敏入库责任方，并说明客户 PDF 已给出部分数据量。
- `docs/other/【写公文】支持公文类型.pdf`：列出支持文种、子类、模板与示例。当前 PDF 中可抽取到的明确数量只出现在“方案”子类：工作方案 1815、活动方案 740、整治方案 315、调研方案 8、会议方案 3。

## 盘点结论

- 当前可证明样本充足的范围只有“方案”下的工作方案、活动方案、整治方案三个子类；“方案”整体应标为部分充足，而非全子类充足。
- 其余 8 个高频文种在仓库材料中未登记明确样本数量。按本盘点口径，不能把模板 / 示例数量当作客户存量范文数量，因此均视为“数量未登记、当前不足以作为首批灰度充足文种”。
- 灰度放开顺序建议以“已完成原文到位 + 清洗 / 脱敏入库 + c03 可检索”为前置条件：先验证工作方案、活动方案、整治方案；其它文种待客户补齐数量、格式、到位时间和责任方后再进入灰度候选。
- 该结论不改变 c05 已实现的结构契约与生成编排能力，也不向 c05 引入文种能力档 / 标黄稀缺配置。c06 仍是文种 / 子类能力档与降级判定的单一权威。

## 待客户 / 实施侧补齐字段

- 每个文种及代表子类的实际可提供范文篇数。
- 原文格式（docx / PDF / 扫描件 / 其它）与是否可机器解析。
- 原文到位时间，是否卡住 Milestone 2 或 UAT。
- 清洗责任方、脱敏责任方、c03 入库责任方与验收人。
- 入库后 c03 可检索样例数，用于后续 7.1 / 7.2 / 7.3 的评测、灰度与 TopK 校准。
