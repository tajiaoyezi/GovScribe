## 盘点口径

- 范围：c05 `c05-high-freq-doctypes` 覆盖的 9 个高频文种：通知、请示、报告、函、会议纪要、通报、批复、讲话稿、方案。
- 数量口径：仅采信仓库内已提交材料中明确写出的范文数量；模板条目、标题示例、链接示例不折算为样本量。
- 充足线：对照 PRD 风险口径，`<100` 条为稀缺线；每文种宜 `>=100` 篇才具备深做灰度底气。若只登记到子类数量，则按子类分别判断，文种整体标为“部分充足”。
- 边界：本盘点仅作为 c06 文种分流与 c05 灰度放开顺序输入；不在 c05 写入文种 / 子类能力档、不判定是否标黄、不替代 c06 分级表。

- 用户于 2026-06-20 提供本地候选素材目录 `各类文件/`：包含类似公文原文及附件。该目录可能含客户 / 单位原文，按安全边界不提交到 git；本文件仅登记脱敏前的数量级与可抽取性汇总，不记录原文正文。

## 盘点结论

- 当前可证明样本充足的范围只有“方案”下的工作方案、活动方案、整治方案三个子类；“方案”整体应标为部分充足，而非全子类充足。
- PDF 材料未登记其余高频文种的明确样本量；2026-06-20 本地候选已覆盖通知、请示、报告、函、会议纪要、通报、批复 7 个文种，但均未达 `>=100` 充足线且未完成清洗 / 脱敏 / c03 入库，因此仍不得作为首批灰度充足文种。
- 2026-06-20 新增本地候选素材可缓解“真实 / 相似公文素材”缺口，但尚未完成清洗、脱敏、c03 入库与目标模型实测，不能直接作为 7.3 的校准结论或 UAT 采纳率证据。
- 灰度放开顺序建议以“已完成原文到位 + 清洗 / 脱敏入库 + c03 可检索”为前置条件：先验证工作方案、活动方案、整治方案；其它文种待客户补齐数量、格式、到位时间和责任方后再进入灰度候选。
- 该结论不改变 c05 已实现的结构契约与生成编排能力，也不向 c05 引入文种能力档 / 标黄稀缺配置。c06 仍是文种 / 子类能力档与降级判定的单一权威。

### 7.3 仍需补齐的证据

- 原文清洗 / 脱敏结果及 c03 可检索记录：确认模型 few-shot 只能取 c03 返回样例，不直接读取本地原文目录。
- 目标模型实测记录：至少覆盖不同 TopK 与提示总长组合，并保留模型、参数、运行时间与输出引用。
- 人工 rubric 与采纳标签：按文种规范、结构完整、行文关系、机关口径四维打分，并按直接用 / 小改 / 大改 / 弃用统计采纳率。
- 速度依据：记录首包耗时、总生成耗时、输出长度与失败 / 超时情况。
- 灰度表同步：通知、请示、报告、函、会议纪要、通报、批复已在灰度批次中引用本地候选 raw/readable 数量，但仍全部处于“待清洗 / 脱敏 / c03 可检索”状态，不得视为试点放开或 7.3 完成证据。
- 缺口补齐：讲话稿、方案原文位置需要补充；通报样本数量需要扩充，否则不能支撑该文种的稳定校准。
