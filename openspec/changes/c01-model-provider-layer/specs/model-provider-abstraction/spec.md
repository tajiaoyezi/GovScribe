## ADDED Requirements

### Requirement: 统一模型供应商窄抽象接口

系统必须（MUST）提供一层 Go 窄抽象接口（对齐 ADR-0001 D1）作为上层业务调用大模型的唯一入口；上层业务（脱敏网关、密级路由、文种生成、RAG 编排）应当只依赖该接口，必须不直接 import 任何具体供应商 SDK（如 `openai-go`、`anthropic-sdk-go`）或 LiteLLM 客户端类型。接口必须同时覆盖两种调用形态：一次性文本生成（chat completion）与流式文本生成；二者的请求参数与返回语义在不同后端实现之间必须保持等价。

#### Scenario: 上层包不出现具体 SDK 导入

- **WHEN** 对上层业务包（脱敏、密级路由、文种生成、RAG 编排）执行依赖扫描
- **THEN** 这些包的导入图中不得出现 `github.com/openai/openai-go`、`github.com/anthropics/anthropic-sdk-go` 或任何 LiteLLM 专有客户端类型
- **AND** 它们对模型的全部调用都必须经过该窄抽象接口

#### Scenario: 一次性文本生成

- **WHEN** 上层通过抽象接口发起一次非流式 chat completion 调用，并传入消息列表与生成参数
- **THEN** 接口必须返回完整生成文本及结束原因（如正常结束、达到长度上限、被内容策略拦截）
- **AND** 返回结构必须与具体后端无关，调用方无需知道底层是 LiteLLM 还是 Go 直连 SDK

### Requirement: 后端实现可替换契约

系统必须（MUST）为该抽象接口提供两个可互换的后端实现：MVP 阶段经 LiteLLM Proxy 统一对接 OpenAI / Anthropic / 私有 OpenAI 兼容端点的后端，信创 / 私有化阶段 Go 直连官方 SDK 的后端（`openai-go` 导入路径必须带 `/v3`，`anthropic-sdk-go` 需 Go 1.24+，版本与许可一律以 ADR-0001 D3 及技术栈版本表为准）。两个后端对同一抽象方法的可观察行为必须等价，使 MVP→信创的后端替换成为配置项而非业务代码改造。

#### Scenario: 切换后端不改业务代码

- **WHEN** 部署形态从 MVP（LiteLLM 后端）切换为信创（Go 直连 SDK 后端）
- **THEN** 上层业务代码必须无需任何改动即可继续工作
- **AND** 切换只通过后端选择配置项完成，不涉及对调用方接口签名的修改

#### Scenario: 两后端行为等价

- **WHEN** 用同一组消息与参数分别经 LiteLLM 后端与 Go 直连后端发起调用
- **THEN** 两者返回的结果结构、结束原因取值集合、错误归因分类必须一致
- **AND** 调用方据返回结构编写的处理逻辑在两种后端下都必须成立

### Requirement: 启动期后端选择

系统必须（MUST）在启动期根据配置项选定当前后端实现（LiteLLM 后端或 Go 直连后端），该选择随发布事件确定、非运行时热切；运行时热切仅适用于模型配置（见 model-config-management），不适用于后端选择。后端选择必须与具体模型配置解耦：后端决定"以何种通道发起调用"，模型配置决定"调用哪个供应商的哪个模型"。

#### Scenario: 启动期通过配置项选定后端

- **WHEN** 应用启动时读取后端选择配置项
- **THEN** 系统必须按配置实例化对应后端实现并将其注入抽象接口
- **AND** 若配置项缺失或取值非法，系统必须拒绝以不确定状态启动并给出明确错误，而不得静默回退到任意默认后端

#### Scenario: 后端选择不在运行时热切

- **WHEN** 应用已按启动期配置选定某后端并运行中
- **THEN** 切换后端实现必须经修改启动配置项并重启应用层（随发布事件）完成，运行时不得热切后端
- **AND** 运行时热切只作用于模型配置（换 key / 换 model / 切公网或私有配置），不改变已选定的后端实现

### Requirement: 暴露当前供应商公网 / 私有属性

抽象接口必须（MUST）把"当前生效供应商是公网还是私有"作为可读状态向上层（特别是 c02 密级路由与 fail-closed 决策）暴露。本 change 不做密级路由判定，但必须保证该状态可被可靠读取，使上层能据此判定数据外发边界。

#### Scenario: 读取当前供应商网络属性

- **WHEN** 上层在发起生成前查询当前生效供应商的网络属性
- **THEN** 接口必须返回明确的"公网"或"私有"标识
- **AND** 该标识必须与当前生效的模型配置（见 model-config-management）保持一致，不得返回未知或模糊状态

### Requirement: 按请求定向到指定供应商配置

抽象接口必须（MUST）支持上层（特别是 c02 的"转私有"）在单次调用粒度上把该请求定向到一条已启用的指定供应商配置（尤其是私有配置），且该定向必须不改变全局当前生效配置（不触发 model-config-management 的运行时切换、不影响其它请求）。本 change 只提供"按请求定向"这一出站能力，不做密级判定与路由决策（属 c02）。当 MVP 阶段不存在任何已启用的私有配置时，"转私有"在本层不可达，接口必须以明确的"无可用私有配置"信号拒绝该定向请求而非静默回退到公网，由上层 c02 据此按 ADR-0001 D7 走降级（仅 regex + 字典、需该密级显式开启）或阻断；涉密永不降级的红线由 c02 执行。

#### Scenario: 单请求定向到指定私有配置不改全局

- **WHEN** 上层为某次调用指定一条已启用的私有供应商配置并经抽象接口发起请求
- **THEN** 该请求必须经指定的私有配置出站
- **AND** 全局当前生效配置必须保持不变，其它并发或后续未指定定向的请求仍走全局当前生效配置

#### Scenario: 无可用私有配置时定向不可达

- **WHEN** 上层请求定向到私有配置，但当前不存在任何已启用的私有供应商配置
- **THEN** 接口必须返回明确的"无可用私有配置"信号并拒绝该定向，而不得静默回退到公网配置出站
- **AND** 本层不替上层做降级或阻断决策，由 c02 据该信号按 ADR-0001 D7 自行降级或阻断（涉密永不降级）

### Requirement: 按请求承载出站内容密级供 c02 拦截点读取

窄抽象调用入参必须（MUST）新增一项**出站请求内容密级（取值：非密 / 敏感 / 涉密；允许缺省为「缺失/未知」）**，其值即 c06 结构化场景上下文契约的「内容密级」项。c01 必须（MUST）原样承载该值并在公网分支上向 c02 拦截点透出；c01 自身不解读、不判定、不改写该密级，也不得将其映射进后端 SDK 的生成参数。调用方未传该值时，c01 必须（MUST）如实呈现「缺失/未知」、不补默认值，使 c02 能据以触发涉密 fail-closed。本层只承载并透传该密级，不做任何密级路由判定（属 c02）。

#### Scenario: 出站内容密级原样透出供 c02 读取

- **WHEN** 上层经窄抽象接口发起调用并在入参中传入出站请求内容密级（取值：非密 / 敏感 / 涉密）
- **THEN** c01 必须原样承载该值并在公网分支上向 c02 拦截点透出，c02 拦截点能从该调用读到该密级
- **AND** c01 不得解读、判定或改写该密级，也不得将其映射进后端 SDK 的生成参数

#### Scenario: 调用方未传出站内容密级时呈现缺失/未知

- **WHEN** 上层经窄抽象接口发起调用但未传出站请求内容密级
- **THEN** c01 必须如实把该项呈现为「缺失/未知」而不补任何默认值
- **AND** c02 拦截点必须能据此区分"已传入 vs 缺失/未知"并触发涉密 fail-closed
