## Why

GovScribe 公网提质走 RAG，必然把客户公文片段送往 OpenAI / Anthropic 等公网模型——这直接撞上体制内单位的保密红线：真实公文里的人名 / 单位 / 文号 / 金额 / 项目代号一旦原文出域，代价极高（PRD Risks「保密红线」）。仅靠提示约束或事后审查都不可靠，必须在调用链上插入一道**强制、可审计、按密级生效**的拦截关口。

更关键的是失败模式：负责抓字典未覆盖新实体的外置 NER 是唯一会"挂"的环节，若它不可用时网关静默放行原文，等于在最危险的时刻泄露最难识别的敏感信息。因此本 change 在 c01 模型接入调用链上插入发送前脱敏网关，以 fail-closed 为底线、按密级路由为骨架，把"非密敏感数据出域"风险降到工程可控范围（定位为风险缓解手段，非"涉密上公网"的合规承诺）。属 MVP 范围（Delivery Milestone 1），**触及保密红线 / 密级路由 / 脱敏**。

## What Changes

- **在 c01 调用链上插入发送前拦截**：所有面向公网模型的调用先过脱敏网关，再决定放行 / 改写 / 改道 / 阻断；映射表、原文、diff、审计全部留在 Go 网关进程内，外置 NER 只收文本片段、只回实体 span。
- **三层敏感实体识别**：① 正则规则（文号 / 金额 / 证件号等格式化实体）；② 客户可维护字典 Aho-Corasick 多模匹配（单位 / 人员名册、项目代号、涉密关键词黑名单）；③ 外置中文 NER 兜底（无状态微服务）抓字典未覆盖的新实体（人名 / 地名 / 机构等）。
- **占位替换 + 本地回填**：命中实体替换为类型化占位符（保留指代一致性，避免脱敏过度丢失单位口吻 / 称谓），模型返回后在本地按映射表回填原值；**映射表绝不出域**。
- **流式回填尾缓冲**：SSE 流式输出场景用 tail-buffer 处理跨 chunk 截断的占位符，防止占位符被拆碎导致回填失败或原值泄露。
- **fail-closed + 按密级降级**：判定 NER "不可用"从宽（超时 / 连接失败 / 5xx / 响应格式异常均算"脱敏未完成"）；处置优先级 ① 自动转私有 / 本地模型 → ② 无私有可用时降级发送（仅 regex + 字典、跳过 NER，**按密级可配开关**，MVP 公网阶段为主回退路径）→ ③ 否则阻断公网调用并明确报错。**涉密永不降级、恒走私有**；fail-open 仅在客户为某密级显式开启时生效，绝非默认。
- **熔断器**：对外置 NER 接熔断器，避免逐请求干等超时拖死写作流，熔断期间直接走既定回退路径。
- **按密级路由**：非密直接走公网、敏感走脱敏后公网、涉密强制走私有化模型；密级 → 模型路由的映射表不出域。
- **客户可维护脱敏库**：提供脱敏库的预导入与增删入口（单位 / 人员名册、项目代号、涉密关键词黑名单），落 PostgreSQL，作为字典层与精确召回的统一词条来源。
- **出公网审计留痕**：每次公网调用记录脱敏前后 diff + 命中项明细 + 降级 / 阻断 / 转私有的处置事件，供合规复核（审计员只读）。

## Capabilities

### New Capabilities

- `desensitization-gateway`: 发送前脱敏网关核心——在公网模型调用链上拦截，执行三层识别（正则 + 字典 AC + 外置 NER）、类型化占位替换与本地回填、流式尾缓冲，并落实 fail-closed 失败模式（NER 不可用时按优先级转私有 / 降级 / 阻断，涉密永不降级）与外置 NER 熔断；映射表 / 原文 / diff 不出 Go 进程。
- `security-classification-routing`: 密级路由——按数据密级（非密 / 敏感 / 涉密）决定调用路径（公网 / 脱敏后公网 / 强制私有化模型），与脱敏网关协同把"涉密"与"NER 不可用且无私有"等高风险场景导向不出域路径；密级 → 模型映射不出域。
- `desensitization-dictionary`: 客户可维护脱敏库——单位 / 人员名册、项目代号、涉密关键词黑名单的预导入与增删管理，落 PostgreSQL，作为字典识别层与（c03）文号 / 单位名结构化精确召回的统一权威词条来源。

### Modified Capabilities

（无：`openspec/specs/` 当前为空，本 change 不修改既有 capability 的 spec 级行为。）

## Impact

- **代码 / 模块**：Go 后端新增脱敏网关模块（识别引擎、映射表管理、占位 / 回填、流式尾缓冲、熔断器、密级路由决策器、脱敏库 CRUD）；新增外置中文 NER Python 微服务（无状态，HanLP / PP-UIE / GLiNER 实测后选，见 ADR-0001 D7）。
- **调用链 / 接口**：在 c01 模型接入调用链（公网分支）上插入网关拦截点；新增脱敏库维护 API、审计查询 API；SSE 流式输出路径接入尾缓冲回填。
- **数据 / 存储**：PostgreSQL 新增脱敏字典表、密级路由配置、脱敏审计日志（diff + 命中项 + 处置事件）。
- **依赖**：前序依赖 c01（模型接入调用链与私有 / 公网供应商抽象，转私有需其支持）；依赖 c04 提供访问控制——脱敏库维护入口消费 c04 `rbac-authorization`（`dict.manage` 权限点）做 RBAC 校验，脱敏审计查询「审计员只读」消费 c04 `rbac-authorization`（`audit.read` 权限点），脱敏审计表「同等密级 ACL」消费 c04 `access-control-composition`（对接 Postgres 预过滤为权威 ACL）；本 change 不自建 RBAC / 密级 ACL 判定。Go 侧引入 Aho-Corasick 库（`petar-dambovaliev/aho-corasick` 锁 commit 或保守 `cloudflare/ahocorasick`，做成可替换接口，见 ADR-0001 D7）；外置 NER 经 HTTP 调用。
- **下游复用**：c03 范文入库复用本 change 的抽取（正则 + 字典）与 `desensitization-dictionary` 词条能力做脱敏入库与结构化精确召回字段抽取。
- **红线 / 范围**：触及保密红线、密级路由、脱敏；落在 MVP 范围（Delivery Milestone 1）。私有化部署形态本身仍后置（Out of scope），本 change 只保证"涉密 / 高风险导向私有路径"的路由与 fail-closed 行为成立。
