## 3. 类型化占位替换与本地回填（D-3）

- [x] 3.1 设计类型化占位符格式：类型 + 单次请求内自增序号（形如 `〖PERSON_01〗`），边界标记选用公文正文罕见且与文号六角括号区分的成对全角符号 `〖…〗`
- [x] 3.2 实现单次请求内存映射表「占位符 ↔ 原值 + 类型 + 命中来源」，同一原值在单次请求内映射到同一占位符以保留指代一致性；映射表绝不出域、不落盘、不传外置 NER
- [x] 3.3 实现占位替换：按命中项集合将原文实体替换为对应类型化占位符
- [x] 3.4 实现本地回填：模型返回后按本次请求映射表把占位符还原为原值；只回填本次请求映射表内登记的占位符，未登记的相似串不回填（防伪占位符碰撞），回填后再交付下游
- [x] 3.5 验证：同一单位名多处替换为同一占位符且回填一致；含占位符的模型返回被正确还原且映射表不出域（覆盖 desensitization-gateway「同一实体保持指代一致」「返回结果本地回填还原」场景）

## 4. 拦截点接入 c01 公网分支与密级路由（D-1 / D-7）

- [x] 4.1 实现网关拦截装饰器，包裹 c01 模型接入窄抽象「目标解析为公网模型」的出站调用，构成 Go 进程内唯一、不可旁路的公网出站收口；私有 / 本地模型目标直接放行不强制脱敏
- [x] 4.2 实现密级路由决策器：读取 c01 窄抽象调用承载的『出站请求内容密级』字段（取值：非密 / 敏感 / 涉密；允许缺省为「缺失/未知」；由 c06 经 c05 / c07 确定、c01 原样承载不解读，与三库密级体系同一口径，不另立判定），非密直走公网（仍过三层识别处置、不强制改道）、敏感脱敏后公网、涉密强制私有
- [x] 4.3 实现内容密级缺失 / 未知兜底：当 c01 窄抽象调用承载的『出站请求内容密级』字段缺失或未知（c01 如实呈现「缺失/未知」、不补默认值）时，按涉密 fail-closed 处置——强制转私有，无可用私有则阻断，涉密永不降级，并入审计；不得缺省按非密放行公网
- [x] 4.4 实现涉密硬约束：禁止任何配置开关使涉密内容走公网或脱敏后公网，涉密恒导向不出域私有路径
- [x] 4.5 实现密级路由配置加载与维护（系统管理员可改密级 → 模型路由及降级开关），配置落 Postgres、不出 Go 进程边界，变更后续请求即时生效
- [x] 4.6 验证：公网调用必经网关、未经处置的公网出站被拒绝；非密直走 / 敏感脱敏后走 / 涉密强制私有；内容密级缺失 / 未知按涉密 fail-closed（强制转私有，无私有则阻断）；私有模型目标不强制脱敏（覆盖 desensitization-gateway「公网调用必经网关处置」「私有 / 本地模型调用不强制脱敏」、security-classification-routing「非密走公网」「敏感走脱敏后公网」「涉密强制私有」「内容密级缺失或未知按涉密 fail-closed」「管理员修改路由配置」场景）

## 5. 外置 NER 兜底、熔断与 fail-closed 状态机（D-2 第三层 / D-5 / D-6）

- [x] 5.1 约定外置 NER 微服务接口契约：无状态、只收文本片段、只回实体 span（人名 / 地名 / 机构等类型），绝不接触映射表与原文落盘（ADR-0001 D7 铁律）
- [x] 5.2 实现 Go 侧外置 NER HTTP 客户端，接入识别流水线第三层，承担字典未覆盖的新实体兜底
- [x] 5.3 实现「NER 不可用」从宽判定：超时 / 连接失败 / 5xx / 响应格式异常（HTTP 200 但 span 格式不符约定）均判为「脱敏未完成」，不只判进程死活
- [x] 5.4 实现 fail-closed 处置状态机，处置优先级：① 存在可用私有 / 本地模型则自动转私有（重解析出站目标）；② 无私有可用且该非涉密密级显式开启降级开关 → 以 regex + 字典结果脱敏后发公网、跳过 NER；③ 否则阻断公网调用并返回明确错误，绝不发原文
- [x] 5.5 固化涉密铁律：涉密恒走私有、永不降级，即使降级开关被开启也不对涉密生效；降级开关默认关闭，仅客户为某非涉密密级显式开启后方对该密级生效
- [x] 5.6 实现外置 NER 熔断器（含半开探测）：连续失败达阈值则打开，打开期间后续请求直接进入 fail-closed 回退路径而不逐请求干等超时；半开探测成功后闭合恢复；阈值 / 超时 / 半开窗口设为可配，先给保守默认
- [x] 5.7 实现路由与网关协同：NER 不可用且无私有且未开降级 → 协同阻断而非回退发原文；涉密在 NER 不可用时仍强制走私有，不因 NER 状态改变涉密路由结论；转私有定向成功发起但私有端点运行时调用失败（c01 返回端点不可达 / 超时 / 上游报错）→ 按 fail-closed 收敛为阻断（涉密绝不回退公网、非密不静默降级原文）并入审计，与「无已启用私有配置」信号为两条不同入口、同归阻断
- [x] 5.8 验证：NER 超时有私有则转私有；无私有已开降级则降级发送；未开降级则阻断；涉密永不降级；响应格式异常视同脱敏未完成；连续失败触发熔断、半开恢复后闭合；降级开关默认关闭；转私有定向后私有端点运行时调用失败收敛为阻断（覆盖 desensitization-gateway「NER 超时且有私有模型可用」「NER 不可用且无私有模型、非密已开启降级」「NER 不可用且未开启降级」「涉密内容永不降级」「响应格式异常视同脱敏未完成」「连续失败触发熔断」「半开探测恢复后闭合」、security-classification-routing「NER 不可用且无私有触发阻断」「涉密在 NER 不可用时仍走私有」「转私有定向运行时失败收敛为阻断」「降级开关默认关闭」场景；对应 `TestHTTPNERClientTreatsUnavailableAndBadResponsesAsUnavailable` / `TestDecoratorTurnsPrivateWhenNERUnavailableAndPrivateExists` / `TestDecoratorDegradesWithRegexAndDictionaryWhenNERUnavailableAndAllowed` / `TestDecoratorBlocksWhenNERUnavailableNoPrivateAndNoDegrade` / `TestDecoratorNeverDegradesClassifiedWhenNERUnavailable` / `TestNERCircuitBreakerOpensAfterConsecutiveFailuresAndHalfOpenCloses`）

## 6. 流式回填尾缓冲（D-4）
