## 3. 会话凭证（JWT）与认证中间件

- [x] 3.1 实现会话 JWT 签发：claims 含用户标识与角色信息、设有效期，签名密钥与受众独立于 OnlyOffice 回调 JWT（D8），二者不复用同一 token → 验证：解析签发的 token 可得 principal 标识与角色；与 OnlyOffice 令牌使用不同密钥 / 受众
- [x] 3.2 实现认证中间件：解析 JWT，对过期 / 签名失败 / 缺失一律按未认证拒绝且不执行业务逻辑 → 验证：篡改签名、过期、无 token 三种请求均返回未认证错误且不返回业务数据
- [x] 3.3 实现"停用 / 角色变更即时失效"折中：中间件解析 JWT 后以 `user_id` 回查 Postgres 当前 `is_active` 与角色（D-2），可加进程内短 TTL 缓存但 TTL 远小于停用容忍窗口 → 验证：账号停用后其未过期既有 JWT 再次访问受保护接口被拒
- [x] 3.4 将解析出的统一内部 principal（用户标识 + 角色）注入请求上下文供下游 RBAC 与叠加判定消费 → 验证：受保护 handler 可从上下文取到当前 principal，未认证请求无 principal
