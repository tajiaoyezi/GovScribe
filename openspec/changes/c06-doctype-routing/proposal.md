## Why

体制内写作者（尤其业务部门兼职写公文的人）最怕"选错文种、写得不像"：同一件事用请示还是报告、用函还是通知，决定了称谓、主送、行文关系与口吻，选错则全篇返工。直接要求用户先从 9 个深做文种 + 大量细分子类中手选，对不熟悉文种区分的人是高门槛；而要素（发文单位、主送机关、事由、行文方向等）缺失又会让后续生成"凭空臆造"。本 change 对应 PRD Milestone 3「文种选择引导」：让用户用自然语言描述写作场景，由 Agent 判别文种 / 子类并正确路由，要素不足时澄清式追问补齐，把"选对文种 + 凑齐要素"的负担从用户转移到 Agent，为后续高质量生成兜住入口。

## What Changes

- 新增**场景描述入口**：用户以自然语言陈述写作意图（如"想给上级单位申请一笔活动经费"），无需预先选定文种。
- 新增**文种 / 子类判别**：基于 c01 模型层做意图分类，依「文种覆盖矩阵」将场景映射到文种与代表子类（如请示 → 资金费用申请），输出**候选文种 + 置信度**；低置信或多义时给出 Top-N 候选供用户一键确认，而非静默猜测。
- 新增**路由分流**：判别命中 9 个深做文种（A 表）时路由到 c05 高频文种生成；落在 B 表其余文种、稀缺子类或无法归类时路由到 c07 通用兜底，**全程不出现"无法处理"**。c06 拥有并在路由阶段查「文种能力档分级表」（owner=c06、来源 PRD 文种覆盖矩阵），对标黄稀缺（<100 条）子类**一次性**判定降级移交 c07，移交判定只发生在 c06。路由结果含目标 capability、文种、子类、行文方向（上行 / 下行 / 平行）等结构化标签。
- 新增**要素澄清式追问**：按目标文种的必需要素清单（发文 / 主送单位、事由、关键事项、时间地点等）校验已知信息，缺关键要素时发起**有限轮次**的澄清提问补齐，要素齐备或用户跳过后再放行至生成。
- 路由与澄清产出的**结构化场景上下文**作为契约交给 c05 / c07，避免下游重复判别。
- 不涉及保密红线变更：本 change 只处理用户输入的场景描述文本与要素，**不调用公网模型生成正文、不触碰范文库检索**；意图分类若需调模型，经 c01 统一模型调用入口透传至 c02 的 desensitization-gateway + security-classification-routing 处置，本 change 不另立脱敏 / 密级路由口径。

## Capabilities

### New Capabilities
- `doctype-classification`: 将用户自然语言场景描述判别为文种 / 细分子类并给出置信度，依「文种覆盖矩阵」分流路由到 c05 深做生成或 c07 通用兜底；低置信 / 多义时返回 Top-N 候选供确认，产出结构化场景上下文（文种、子类、行文方向）。
- `slot-clarification`: 依目标文种的必需要素清单校验场景信息完整度，缺关键要素时发起有限轮次澄清式追问补齐，要素齐备或用户显式跳过后放行至下游生成。

## Impact

- **新增代码模块**：文种路由服务（场景输入 → 判别 → 分流 → 要素校验 → 移交），位于 Go 应用层；判别提示词 / 文种与子类映射规则、必需要素清单作为可维护配置。
- **依赖前序 change**：c01（意图分类经统一模型调用入口接入，提供窄抽象供应商定向能力）、c02（脱敏网关 desensitization-gateway + 出站密级路由 security-classification-routing，对判别调用的敏感 / 涉密内容做处置）、c05（深做文种生成为路由命中目标）、c07（通用兜底为路由兜底目标）。
- **对下游接口**：定义"结构化场景上下文"契约（字段固定为 9 项：目标 capability、目标文种、代表子类、行文方向、置信度、用户原始场景描述、已补齐要素、缺失 / 未确认要素标记、内容密级（非密 / 敏感 / 涉密）），由 c05、c07 消费，三者需就字段对齐；内容密级由写作发起方在发起时确定、定级策略归客户密级策略，随契约移交并作为出站请求密级随 c01 调用透传供 c02 路由，缺失按涉密 fail-closed。
- **前端（React + Ant Design）**：写作工作台新增场景描述入口、候选文种确认与澄清追问交互；判别 / 澄清过程不要求流式，正文生成的 SSE 流仍属 c05 / c07。
- **角色**：面向 D10 中的「业务兼职用户」（文种引导 + 通用兜底起草）与「文秘」核心场景；受现有 RBAC 约束，本 change 不新增权限模型。
- **MVP 范围内**：属 PRD Milestone 3 已确认范围；不触及版式（GB/T 9704）、OA 集成、智能核稿等后置项。
