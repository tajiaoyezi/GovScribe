## Why

高频文种与通用兜底已能产出"像公文"的初稿（c05/c07），但初稿仍是草稿——文秘必须能在浏览器内逐字核稿、按机关口径修订、决定是否采纳，并把成稿带走精排，否则"辅助起草"无法闭环到"可直接采纳"。PRD 把"在线编辑/核稿界面"与"人工最终把关"列为 MVP 纳入项：Agent 不自动发文，所有产出由人工评判、决定采纳。同时，采纳决策（直接用/小改/大改/弃用）既是主指标"初稿采纳率"的数据来源，也是公网提质路线里"采纳稿回流"的入口，缺了它 UAT 无从量化、RAG 无可回流的优质样例。本 change 交付这块编辑/核稿/采纳的人工把关界面。

## What Changes

- 集成 OnlyOffice Document Server 社区版作为公文在线编辑/核稿界面：docx 原生保真 + 实时协同；与 Go 后端经 JWT 回调集成，文档读写走 MinIO 预签名 URL（口径遵 ADR-0001 D8，社区版 AGPL v3、禁去品牌、不改其源码仅经 API/JS 集成）。
- 文秘可在浏览器内打开 AI 初稿进行编辑与核稿，对照机关口径逐处修订并保存为生成稿版本。
- 提供"采纳决策"动作：每篇初稿记录直接用/小改/大改/弃用四档结论，作为采纳率口径的数据来源；被采纳（直接用/小改）的成稿触发回流——本 change 产出字段固定为「成稿引用（MinIO 对象键/chunk 标识）+ 采纳结论 + 密级 + 文种」的回流决策信号（密级继承原生成任务或采纳/核稿时文秘确认），经与 c03 约定的同一 outbox/事件表交付，实际入库由 c03 单点负责，本 change 不直调 API、不直接入库。
- 导出能力：把成稿导出为 .docx 供文秘带走精排；导出做成抽象 `export(docx)`，预留 `export(ofd)` 接口位但 MVP 不实现（OFD 暂不支持，遵 D9）。
- 编辑器做成可替换适配层：前端编辑组件 + 后端回调/导出接口对业务封装，内容按公文语义结构（标题、主送、正文结构、落款等）结构化存储、与版式解耦，便于信创终局换 WPS / 接 OFD 服务时业务代码不动（遵 §2「编辑器可替换」）。

不在本 change 范围（属其它 change）：初稿生成（c05/c07）、采纳稿向量入库与 RAG 回流落地（c03）、谁能编辑/谁能采纳的权限判定（c04）、GB/T 9704 红头版式精排与 OFD 输出（后置，遵 D9）。

保密红线/密级路由/脱敏：本 change 是浏览器内人工编辑与导出，不触发对公网模型的调用，因此不直接经过脱敏网关与密级路由；但成稿落 MinIO 生成稿桶、决策与元数据落 PostgreSQL，须沿用 c04 的密级/RBAC 叠加判定约束（能否打开/编辑/采纳/导出由其裁定，本 change 调用其判定而不另立口径）。落在 MVP 范围内。

## Capabilities

### New Capabilities

- `online-document-editing`: 浏览器内公文在线编辑/核稿能力与编辑器可替换适配层——OnlyOffice 社区版接入（JWT 回调鉴权、MinIO 预签名 URL 读写、docx 保真与协同）、前端编辑组件与后端回调接口的抽象边界、成稿按公文语义结构化存储且与版式解耦的持久化契约。
- `document-export`: 成稿导出能力与导出抽象——MVP 实现 `export(docx)`（纯 Go 导出库选型遵 D8），统一导出接口预留 `export(ofd)` 接口位（MVP 不实现，遵 D9），保证编辑器/导出后端可替换。
- `review-adoption-decision`: 核稿采纳决策的记录与回流触发——每篇初稿登记直接用/小改/大改/弃用四档结论及成稿引用，作为初稿采纳率口径的权威数据来源，并对被采纳成稿产出供 c03 回流入库的决策信号。

### Modified Capabilities

（无：本 change 仅引入新 capability，不修改既有 spec 级行为。）

## Impact

- 新增后端：OnlyOffice JWT 回调接口（文档打开/保存/状态变更）、MinIO 预签名 URL 签发、成稿版本与公文语义结构持久化、导出接口 `export(docx)`（预留 `export(ofd)`）、采纳决策登记接口。
- 新增前端：写作工作台内的在线编辑/核稿组件（React + TypeScript + Ant Design 中嵌入 OnlyOffice 编辑器）、采纳决策四档交互、导出入口。
- 依赖与集成：依赖 c05/c07 产出的初稿作为编辑输入；以 c04 登记的权限点 `document.open` / `document.edit` / `document.export` / `review.online` / `adopt.decide` 调用 c04 的 RBAC + 密级叠加判定裁定打开/编辑/导出/核稿/采纳权限；向 c03 经约定 outbox/事件表提供被采纳成稿与回流决策信号以供单点入库。
- 存储/外部组件：PostgreSQL（成稿元数据、公文语义结构、采纳决策、版本记录）、MinIO（生成稿桶存放成稿原件与导出产物）、新增运行时组件 OnlyOffice Document Server 社区版（AGPL v3、带品牌、不改源码仅 API/JS 集成；正式政企交付的商业授权/去品牌/信创认证为后置成本，遵 D8）。
- 信创/合规遗留项（仅登记，本 change 不解决）：OnlyOffice 统信/龙芯官方认证与 AGPL 法务确认、ARM64/LoongArch64 镜像可用性 PoC（遵 ADR-0001 §5/§6）。
