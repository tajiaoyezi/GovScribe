# access-control-composition

## Purpose

定义 RBAC 操作授权与密级数据 ACL 的统一叠加判定入口，确保受控操作必须同时通过操作权限与数据权限检查，缺失或不可用时按 fail-closed 拒绝。

## Requirements

### Requirement: RBAC 与密级两层正交叠加判定

系统应当（SHALL）提供统一的访问控制判定点，将 **RBAC（管"能做什么操作"）** 与 **密级数据 ACL（管"能看 / 能外发什么数据"，以 PostgreSQL 预过滤为权威 ACL，由后端注入过滤条件）** 两层正交叠加判定（口径见 ADR-0001 §2 / D10 与 PRD 密级防越权）。两层不可相互替代：任一层返回拒绝即整体拒绝，必须（MUST）两层均允许方可整体允许。本判定点的「密级臂」对接 PostgreSQL 预过滤为权威 ACL 裁决数据可见 / 可外发，必须不自建第二套密级判定，必须不重写脱敏 / 出站密级路由的出域逻辑（出域路由属 c02）。

#### Scenario: 两层均允许才整体放行

- **WHEN** RBAC 判定允许某操作且密级数据 ACL（PostgreSQL 预过滤）判定允许对应数据可见 / 可外发
- **THEN** 统一判定点必须返回整体允许

#### Scenario: RBAC 拒绝则整体拒绝

- **WHEN** RBAC 判定拒绝某操作（无论密级层结果如何）
- **THEN** 统一判定点必须返回整体拒绝，不得因密级层允许而放行

#### Scenario: 密级层拒绝则整体拒绝

- **WHEN** RBAC 判定允许某操作，但密级数据 ACL（PostgreSQL 预过滤）判定拒绝对应数据的可见 / 可外发
- **THEN** 统一判定点必须返回整体拒绝，不得因 RBAC 允许而放行

#### Scenario: 两层不可相互替代

- **WHEN** 仅 RBAC 通过而未取得密级判定结果，或仅密级判定通过而未经 RBAC 判定
- **THEN** 系统必须不得视为整体允许，缺失任一层判定时必须按拒绝处理

### Requirement: 统一鉴权决策接口及自身受控操作自校验

系统应当（SHALL）对外提供一致的"是否允许"鉴权决策接口，供 c01–c08 各 change 在执行受控操作前统一调用；该接口的入参应当包含已认证 principal（来自 local-identity-auth）、目标操作权限点与目标数据的密级上下文，出参为整体允许 / 拒绝的决策结果。本 change 自身的受控操作（如用户 / 角色管理）必须（MUST）经该统一判定点自校验后方可执行。「调用方不得各自硬编码权限 / 密级判断、不得绕过该接口直连数据」属对消费方 change 的接口契约约束，其落地与验收在各消费方 change，不作为本 change 的 MUST 验收。

#### Scenario: 业务 change 调用统一接口取得决策

- **WHEN** c03 范文检索 / 入库或 c05–c08 起草、在线核稿 / 采纳在执行受控操作前调用该接口
- **THEN** 系统应当基于 principal、操作权限点与密级上下文返回整体允许 / 拒绝的统一决策，调用方据此放行或拦截

#### Scenario: 管理端操作受统一接口约束

- **WHEN** c01 模型配置（`model.config`）、c02 脱敏库维护（`dict.manage`）或审计只读（`audit.read`）的管理端操作发起请求
- **THEN** 这些操作必须先经统一鉴权决策接口判定，未通过 RBAC 层即被拒绝

#### Scenario: 本 change 自身受控操作经统一判定自校验

- **WHEN** 本 change 自身的用户 / 角色管理等受控操作被发起
- **THEN** 系统必须先经统一鉴权决策接口判定，未通过即拒绝，不得绕过该接口直接执行

### Requirement: 访问控制 fail-closed 缺省拒绝

系统在访问控制判定的任一前置条件缺失或不可用时应当（SHALL）默认拒绝（fail-closed），与脱敏 / 密级红线的 fail-closed 原则一致。当 principal 未认证、操作权限点未声明、或密级数据 ACL（PostgreSQL 预过滤）判定结果不可获取时，统一判定点必须（MUST）返回拒绝，必须不得在判定信息不完整的情况下静默放行。

#### Scenario: 未认证请求被拒绝

- **WHEN** 进入统一判定点的请求未携带有效 principal（未通过 local-identity-auth 认证）
- **THEN** 系统必须返回拒绝，不进入 RBAC 或密级判定

#### Scenario: 密级判定不可用时拒绝

- **WHEN** 密级数据 ACL（PostgreSQL 预过滤）判定信息不完整或因依赖不可用而无法获取
- **THEN** 系统必须按拒绝处理，不得在缺少密级判定的情况下放行涉及数据可见 / 可外发的操作

#### Scenario: 未声明操作权限点时拒绝

- **WHEN** 目标操作没有对应的已声明操作权限点
- **THEN** 系统必须默认拒绝，不得对未受控声明的操作放行
