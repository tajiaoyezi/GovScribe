# desensitization-gateway

## Purpose

定义公网模型调用前的脱敏强制拦截、三层敏感实体识别、占位替换与本地回填、fail-closed 回退、熔断和出公网审计能力，确保敏感内容在出域前被受控处置。

## Requirements

### Requirement: 公网调用前强制拦截

脱敏网关应当（SHALL）在 c01 模型接入调用链的公网分支上插入一道强制拦截点：所有面向公网模型（OpenAI / Anthropic 等）的调用必须（MUST）先经过网关处置（放行 / 改写 / 改道私有 / 阻断）后才允许发出原始报文，不存在绕过网关直达公网的旁路。映射表、原文、脱敏前后 diff 与审计数据必须全部保留在 Go 网关进程内，外置 NER 服务只接收待识别文本片段、只回实体 span，绝不接触映射表与原文落盘。

#### Scenario: 公网调用必经网关处置

- **WHEN** 任一业务模块发起一次目标为公网模型的请求
- **THEN** 网关必须在原文离开 Go 进程前完成识别与处置决策，未经网关处置的公网出站请求应当被拒绝

#### Scenario: 私有 / 本地模型调用不强制脱敏

- **WHEN** 一次调用的目标解析为私有化 / 本地模型
- **THEN** 网关应当放行而不强制执行脱敏改写（数据不出域，无需脱敏）

### Requirement: 三层敏感实体识别

网关应当（SHALL）对待出域文本执行三层识别，且层次互补：① 正则规则识别格式化实体（文号如 `〔2024〕X号`、金额、身份证 / 统一社会信用代码等证件号）；② 客户可维护字典经 Aho-Corasick 多模匹配识别单位 / 人员名册、项目代号与涉密关键词黑名单；③ 外置中文 NER 兜底识别字典未覆盖的新实体（人名 / 地名 / 机构等）。正则层与字典层必须运行在 Go 网关进程内，不依赖外置 NER 的可用性；Aho-Corasick 实现应当封装为可替换接口（候选实现见 ADR-0001 D7）。

#### Scenario: 三层各自命中并合并

- **WHEN** 文本同时包含文号、字典中的单位名与一个字典未收录的人名
- **THEN** 正则命中文号、字典命中单位名、NER 命中人名，三层结果合并为统一命中项集合后进入占位替换

#### Scenario: 正则与字典层不随 NER 宕机失效

- **WHEN** 外置 NER 服务不可用
- **THEN** 正则层与字典层仍应当在 Go 进程内正常识别格式化实体与字典词条

### Requirement: 类型化占位替换与本地回填

网关应当（SHALL）将命中实体替换为类型化占位符（区分人名 / 单位 / 文号 / 金额等类型），同一原值在单次请求内必须映射到同一占位符以保留指代一致性，避免因脱敏过度丢失单位口吻与称谓；占位符设计应当便于完整匹配与回填。模型返回后，网关必须在本地按映射表把占位符回填为原值，映射表绝不出域。

#### Scenario: 同一实体保持指代一致

- **WHEN** 同一单位名在一段文本中出现多次
- **THEN** 各处必须替换为同一类型化占位符，回填后还原为同一原值

#### Scenario: 返回结果本地回填还原

- **WHEN** 公网模型返回包含占位符的文本
- **THEN** 网关必须按本次请求的映射表把占位符还原为原值后再交付下游，且映射表不随请求出域

### Requirement: 流式回填尾缓冲

在 SSE 流式输出场景下，网关应当（SHALL）采用尾部缓冲（tail-buffer）处理跨 chunk 截断的占位符：当某 chunk 末尾可能是被拆碎的占位符前缀时，必须暂缓输出该尾部、与后续 chunk 拼接后再判定与回填，防止占位符被拆碎导致回填失败或原值泄露。

#### Scenario: 占位符跨 chunk 截断

- **WHEN** 一个占位符被流式切分到两个相邻 chunk
- **THEN** 网关必须缓冲前一 chunk 的疑似占位符尾部，待拼接完整后再回填并输出，不得输出半截占位符或泄露原值

#### Scenario: 流结束时刷新尾缓冲

- **WHEN** 流式响应结束且尾缓冲中仍有未决内容
- **THEN** 网关必须在收尾时对尾缓冲做最终回填并输出，不得丢弃缓冲内容

### Requirement: fail-closed 失败模式与按密级降级

网关默认应当（SHALL）fail-closed：外置 NER 不可用时必须（MUST）绝不静默发送原文。对"NER 不可用"的判定必须从宽——超时、连接失败、5xx、响应格式异常均算"脱敏未完成"，不得只判进程死活。处置优先级必须为：① 自动转私有 / 本地模型；② 无私有可用时降级发送（仅 regex + 字典、跳过 NER 兜底），降级为按密级生效的可配置开关，MVP 公网阶段（无私有模型）降级即主回退路径；③ 仍不满足则阻断公网调用并明确报错。涉密内容必须恒走私有、永不降级；fail-open（降级发送）只能是客户为某密级显式开启的结果，绝非默认。每次降级 / 阻断 / 转私有必须入审计。

#### Scenario: NER 超时且有私有模型可用

- **WHEN** 外置 NER 调用超时且当前存在可用私有 / 本地模型
- **THEN** 网关必须自动改道私有模型，不向公网发送，并记录"转私有"审计事件

#### Scenario: NER 不可用且无私有模型、非密已开启降级

- **WHEN** NER 不可用、无私有模型可用，且该非密 / 敏感密级已显式开启降级
- **THEN** 网关应当以 regex + 字典识别结果脱敏后发送公网（跳过 NER），并记录"降级发送"审计事件

#### Scenario: NER 不可用且未开启降级

- **WHEN** NER 不可用、无私有模型可用，且当前密级未开启降级开关
- **THEN** 网关必须阻断该公网调用并向调用方返回明确错误，不得发送原文

#### Scenario: 涉密内容永不降级

- **WHEN** 待处理内容密级为涉密
- **THEN** 网关必须强制走私有路径，禁止降级或发送公网，即使降级开关被开启也不得对涉密生效

#### Scenario: 响应格式异常视同脱敏未完成

- **WHEN** 外置 NER 返回 HTTP 200 但响应体不符合约定的 span 格式
- **THEN** 网关必须判定为"脱敏未完成"并按 fail-closed 处置优先级处理，不得当作识别成功放行

### Requirement: 外置 NER 熔断器

网关应当（SHALL）对外置 NER 调用接入熔断器（circuit breaker），避免在 NER 故障时逐请求干等超时拖死写作流。当失败累积触发熔断后，网关必须在熔断打开期间直接走既定回退路径（按 fail-closed 处置优先级），而非继续逐个请求等待超时；熔断器应当具备半开探测以在 NER 恢复后自动闭合。

#### Scenario: 连续失败触发熔断

- **WHEN** 外置 NER 在短时间内连续失败达到熔断阈值
- **THEN** 熔断器必须打开，后续请求在熔断期间直接进入 fail-closed 回退路径而不再逐请求等待超时

#### Scenario: 半开探测恢复后闭合

- **WHEN** 熔断打开一段时间后进入半开状态且探测请求成功
- **THEN** 熔断器应当闭合并恢复正常调用外置 NER

### Requirement: 出公网审计留痕

网关每次涉及公网调用或处置决策应当（SHALL）落审计到本能力拥有的「脱敏审计表」：记录脱敏前后 diff、命中项明细（类型 / 来源层 / 位置），以及降级 / 阻断 / 转私有等处置事件；账号 / 角色 / 认证 / 授权 / 登录失败类审计不属本表，归 c04「账号安全审计表」。审计供合规复核，审计员只读——「审计员」角色及 `audit.read` 只读授权由 c04 `rbac-authorization` 登记与判定，审计查询入口消费该决策、不自建授权。审计数据落 PostgreSQL、纳入与原文同等密级 ACL（由 c04 `access-control-composition` 判定），与映射表 / 原文一样不出 Go 进程边界对外暴露。

#### Scenario: 脱敏后正常出公网留痕

- **WHEN** 一次请求脱敏后成功发往公网模型
- **THEN** 网关必须记录脱敏前后 diff 与命中项明细到审计日志

#### Scenario: 处置事件留痕

- **WHEN** 网关执行了转私有 / 降级发送 / 阻断中的任一处置
- **THEN** 网关必须记录对应处置事件类型与触发原因到审计日志，供审计员只读查询
