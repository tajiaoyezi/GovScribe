# model-config-management

## Purpose

定义模型接入配置的持久化、生命周期管理、运行时切换与连通性探活要求，确保上层业务始终通过受控配置调用模型端点，并避免敏感凭据泄露。

## Requirements

### Requirement: 模型配置持久化于 PostgreSQL

系统必须（MUST）把模型接入配置作为权威数据落 PostgreSQL（对齐 ADR-0001 D5）。每条配置必须至少包含：供应商类型（OpenAI / Anthropic / 私有 OpenAI 兼容端点）、`base_url`、`api_key`、`model`、网络属性（公网 / 私有）与启用状态。`api_key` 等敏感字段必须妥善保管，必须不以明文写入审计日志或对外接口返回体。

#### Scenario: 新增一条模型配置

- **WHEN** 管理员提交一条含供应商类型、`base_url`、`api_key`、`model` 的新配置
- **THEN** 系统必须将其持久化到 PostgreSQL 并赋予唯一标识
- **AND** 后续对该配置的读取接口返回体中 `api_key` 必须被掩码或省略，必须不回显明文

#### Scenario: 敏感字段不入审计明文

- **WHEN** 任意配置的创建、修改、切换操作产生审计记录
- **THEN** 审计记录中必须不包含 `api_key` 明文
- **AND** 审计记录应当保留足以追溯的非敏感信息（操作者、时间、目标配置标识、动作类型）

### Requirement: 管理员维护配置生命周期

系统必须（MUST）提供管理端接口，允许管理员对模型配置执行增、改、查与启用 / 停用。配置的网络属性（公网 / 私有）必须在创建时确定并可被上层读取（供 c02 密级路由使用）。

#### Scenario: 修改既有配置

- **WHEN** 管理员修改某条配置的 `base_url`、`model` 或 `api_key`
- **THEN** 系统必须更新 PostgreSQL 中的对应记录
- **AND** 若被修改的配置正是当前生效配置，系统对生效语义的处理必须遵循"运行时切换当前生效配置"需求

#### Scenario: 停用配置

- **WHEN** 管理员停用某条配置
- **THEN** 该配置必须不可再被选为当前生效配置
- **AND** 若被停用的配置恰为当前生效配置，系统必须拒绝停用或要求先切换到另一条已启用且通过探活的配置，必须不允许进入"无生效配置"的不确定状态

#### Scenario: 未授权角色访问模型配置入口被拒

- **WHEN** 非系统管理员 / 无 `model.config` 权限点的 principal 调用模型配置入口
- **THEN** 系统必须经 c04 rbac-authorization 判定拒绝该请求，必须不执行任何配置变更
- **AND** 本 change 不自建 RBAC 判定，授权决策一律以 c04 `model.config` 权限点为准

### Requirement: 运行时切换当前生效配置

系统必须（MUST）支持在运行时切换当前生效的模型配置，无需重启应用层。切换语义必须显式吸收 LiteLLM 后端的已核验行为（ADR-0001 D3）：热切换对在途流不生效、多 worker 约 30s 传播延迟、Redis 缓存存在 TTL 陈旧期。系统必须不向管理员承诺"立即对所有在途请求生效"。

#### Scenario: 切换不重启即生效于新请求

- **WHEN** 管理员把当前生效配置从配置 A 切换为已启用的配置 B
- **THEN** 切换之后新发起的生成请求必须使用配置 B
- **AND** 整个过程必须不要求重启应用层

#### Scenario: 在途流不受切换影响

- **WHEN** 在配置 A 的流式生成尚未结束时，管理员切换到配置 B
- **THEN** 该在途流必须继续以配置 A 完成或按既定中断策略结束，必须不在流中途改用配置 B
- **AND** 系统应当向管理员明确告知切换对在途请求不生效，且 LiteLLM 后端下存在最长约 30s 的传播延迟与缓存陈旧期

### Requirement: 配置连通性探活测试

系统必须（MUST）提供连通性测试能力：在保存或切换某条配置前，可对其目标端点发起一次最小探活调用，返回可用性结果与基础错误归因（如鉴权失败、端点不可达、模型名不存在、超时）。探活必须使用最小代价的调用，必须不产生实质业务生成内容。

#### Scenario: 探活通过

- **WHEN** 管理员对一条配置发起连通性测试且目标端点正常响应
- **THEN** 系统必须返回"可用"结果
- **AND** 该结果应当可作为允许将此配置设为当前生效配置的前置条件

#### Scenario: 探活失败给出归因

- **WHEN** 探活调用因鉴权失败、端点不可达、模型名不存在或超时而未成功
- **THEN** 系统必须返回"不可用"结果并附带对应的基础错误归因分类
- **AND** 系统必须不把该不可用配置静默推为当前生效配置
